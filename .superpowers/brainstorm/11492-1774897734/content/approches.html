3 approches pour la stack technique

Toutes tournent sur Cloudflare Pages + Workers + D1 + R2

A

React PWA + Hono + D1 + R2

Frontend : React + Vite (PWA) sur Cloudflare Pages
Backend : Hono.js sur Cloudflare Workers (API REST)
DB : Cloudflare D1 (SQLite)
Fichiers : Cloudflare R2
Auth : JWT custom + magic link email

Avantages

  • Frontend et backend bien séparés
  • React très répandu, facile à maintenir
  • Hono conçu spécifiquement pour CF Workers

Inconvénients

  • Auth entièrement custom à coder
  • Deux projets à gérer (frontend + backend)
B ⭐

SvelteKit + D1 + R2 (Recommandé)

Full-stack : SvelteKit avec adaptateur Cloudflare natif
DB : Cloudflare D1
Fichiers : Cloudflare R2
Auth : better-auth (email/password + magic link)

Avantages

  • Un seul projet — frontend + API dans le même dépôt
  • Adaptateur Cloudflare officiel, déploiement simple
  • better-auth gère magic link out-of-the-box
  • Moins de code à écrire

Inconvénients

  • Svelte moins connu que React
  • Courbe d'apprentissage pour les nouveaux contributeurs React
C

React Router v7 (Remix) + D1 + R2

Full-stack : React Router v7 avec adaptateur Cloudflare
DB : Cloudflare D1
Fichiers : Cloudflare R2
Auth : better-auth ou remix-auth

Avantages

  • React (familier) + full-stack en un seul projet
  • Patterns de chargement de données très propres
  • Adaptateur Cloudflare officiel

Inconvénients

  • React Router v7/Remix en pleine évolution (API instable)
  • Plus verbeux que SvelteKit pour ce type d'app
