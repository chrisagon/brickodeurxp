Modèle de données — Section 2/4

Schéma simplifié des tables D1 (SQLite)

users
  • id
  • email
  • password_hash
  • role (jeune|animateur|parent|admin)
  • nom, prénom
  • created_at
parent_child
  • id
  • parent_id → users
  • child_id → users
magic_links
  • token
  • email
  • expires_at
  • used
domains
  • id
  • name (Brick / Codeur)
  • color
  • icon
skills
  • id
  • domain_id → domains
  • title
  • description
  • level_required (1–5)
  • active
badge_requests
  • id
  • jeune_id → users
  • skill_id → skills
  • status (pending|approved|rejected)
  • proof_url (R2)
  • proof_type (photo|video)
  • submitted_at
  • reviewed_at
  • reviewer_id → users
  • reviewer_comment
badges

Badge attribué = compétence validée

  • id
  • jeune_id → users
  • skill_id → skills
  • request_id → badge_requests
  • awarded_at
  • level (blanc|jaune|orange|rouge|noir)

💡 Le niveau est calculé automatiquement : nb de badges validés dans le domaine → 1=blanc, 2=jaune, 3=orange, 4=rouge, 5=noir
